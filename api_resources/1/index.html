APIs

/apis
Enables developers to create and manage APIs in Apigee Enterprise
In Apigee Enterprise, your API is made up of a base URL and a number of resources. Resources are the assets offered by your API. For example, if your base URL is example.com/v1/, your resources could be something like examples.com/v1/blogs. When you add an API, you're creating a new API that acts as a proxy for an existing API. So instead of calling your existing API, your developers would call the new Apigee API.
