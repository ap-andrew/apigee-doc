Users of Apigee Enterprise
Before your team can start using Apigee Enterprise, you need to have an account, that is, an organization, andand an account within that organization. Your organization was given an Apigee Enterprise account as part of the set up. If you find that you don't have an account, please contact Apigee Support. Support can also help if you need to delete an account. Viewing user data The Users table on the Admin tab lists all of the users attached to the current account. You can see the name of the current account by clicking the Home icon. For each user you can see: Role: The role of the user, which determines the level of access By default, all users have an orgadmin role that gives them full access to all the features in Apigee Enterprise Primary email: The email address you entered when you created the user Last sign in: The date this user last accessed Apigee Enterprise Last modified: The date this user's information was last edited Adding users Users of the management system are members of the API team that will be developing, testing API or running reports as opposed to your developers. Those you can add through the Developer tab or they can add themselves through your developer portal. Click the Admin tab. Click + Add User. Enter the user's first name, last name, email address, and password, and then click OK. Deleting users There are two ways to delete a user: If you want to remove a user from your account, select them in the Users table and click Delete. This only removes the user from the current account. If the user is a member of multiple accounts, they remain in the system. If you want to remove a user from Apigee completely, please contact Apigee Support.
