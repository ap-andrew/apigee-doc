Create an API Product :: /apiproducts

API resource: API Products

Verb: POST

Doc: Use the POST method to create a new API product. You need to: Name the API product Bundle Resources by defining resource paths (URIs) Define the "Service Plan" settings for quota (permitted requests per day/month) and/or spike arrest (permitted requests per second/minute).&nbsp; This defines how much your developers can consume over time. At runtime, these variables are read by the Rate Limiting Policy applied to an API or Resource. Rate Limiting Variabes are set as API Product attributes: developer.quota.limit - integer developer.quota.interval - integer developer.quota.timeunit second/minute/hour/day/month) You can also use attributes to set any arbitrary metadata that you need for monitoring or analytics

Description: Create an API product to enable developers to access a set of API resources (URIs)

cURL: 
curl -u {myname:mypass} "https://api.enterprise.apigee.com/v1/o/{myorg}/apiproducts" -X POST -d'
