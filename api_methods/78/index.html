Get Custom Report :: /environments/{environmentName}/stats/{customVariable}

API resource: Analytics

Verb: GET

Doc: A report in Analytics is made up of dimensions, measures and filters. Dimensions lets you view data from various perspectives. For example, the time dimension show you the breakdown of calls for the last month, day or hour. Throughput dimension help you see, which APIs are the most active. Error rate dimension help you pinpoint the APIs that are causing issues with your servers. Average end point response time dimension help you see how your server infrastructure is performing. Measures are numeric representations of a set of facts that have occurred. Examples of measures include APIs, apps, API Products and developers and so forth. By combining different dimensions and measures you can generate reports that show you the exact data you want to see. Filters let you get a subset of results. For example throughput aggregated by APIs, where APIname falls within a range of names. A filter is expressed as a query param in the APIs and is categorized as either: Simple expressions Group expressions Connectors

Description: Enables the generation of a custom report that incorporates dimensions, measures, and filters.

cURL: 
curl -u myname:mypass https://api.enterprise.apigee.com/v1/o/{myorg}/environments/prod/stats/category?select=sum(price)&timeRange=1%2F1%2F2012%2000%3A00~2%2F10%2F2012%2000%3A00&timeUnit=minute&sortby=sum(price)&sort=DESC'
